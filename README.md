# 📊 ArgentinaIdeal-DWH: Data Warehouse & KPIs Analytics

## 📌 Descripción del Proyecto
Diseño y conceptualización de una solución analítica integral en la nube. Este proyecto documenta la arquitectura lógica de un Data Warehouse y Data Marts específicos para centralizar la información de negocio de "ArgentinaIdeal". Su propósito es establecer las bases estructurales para realizar consultas analíticas escalables, calcular KPIs de ventas y optimizar campañas publicitarias.

## 🎯 Objetivos
- Diseñar la arquitectura lógica de datos enfocada en el rendimiento analítico (modelado estrella/copo de nieve).
- Estructurar la lógica de las consultas (queries) para manejar grandes volúmenes de información.
- Proveer una base sólida orientada a Business Intelligence para el seguimiento de métricas clave de la empresa.

## 🛠️ Tecnologías y Herramientas Utilizadas
- **Base de Datos / Cloud:** Google BigQuery
- **Lenguaje de Consulta:** SQL Avanzado
- **Conceptos Aplicados:** Modelado de Datos, Data Warehousing, Arquitectura OLAP, Business Intelligence.

## 🏗️ Arquitectura y Modelado de Datos
El proyecto se centró en el diseño estructural del negocio, definiendo:
- **Tablas de Hechos (Fact Tables):** Centralización de los eventos clave del negocio, como las transacciones de ventas y el rendimiento de los anuncios.
- **Tablas de Dimensiones (Dimension Tables):** Atributos descriptivos para segmentar la información (ej. Tiempo, Producto, Cliente, Campaña).
- **Lógica de Data Marts:** Definición de estructuras específicas para facilitar la consulta directa por parte de los equipos de Marketing y Ventas.

## 🚀 Resultados del Diseño
- Se definió un esquema optimizado para entornos OLAP que reduce la redundancia y prioriza la velocidad de lectura.
- Se establecieron las reglas de negocio y la lógica de las consultas SQL necesarias para calcular métricas críticas como el ROI publicitario, el volumen de ventas por región y el ticket promedio.
